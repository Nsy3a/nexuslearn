# 手动端到端验证步骤

## 前置条件
- 后端：http://localhost:8080 已启动
- 前端：http://localhost:3000 已启动
- 链：本地 Hardhat 节点运行，合约已部署

## 验证流程

### 1. 学习闭环
1. 打开 http://localhost:3000
2. 点击“学习”
3. 输入用户ID：u-demo
4. 目标：CPA
5. 技能：会计基础,练习题,复盘
6. 点击“创建目标”→提示成功
7. 点击“生成计划”→出现7天计划时间线
8. 正确率输入0.9，点击“提交评估”→提示“下一难度：高”
9. 图表区域出现进度柱状图

### 2. 知识仓库与搜索
1. 点击“搜索”
2. URL输入：https://example.com/cpa
3. 点击“录入URL”→提示已录入
4. 关键词输入：会计，点击“搜索”→出现结果列表与Rank
5. 点击“列出资料”→表格展示资料元数据

### 3. 需求协作与链上闭环
1. 点击“协作”
2. 标题：生成前端登录页
3. 预算：100
4. 技术栈：Vue3,TypeScript
5. 点击“创建需求”→提示需求已创建（链上Escrow同步创建）
6. 答案URI输入：ipfs://QmAnswer
7. 点击“提交答案”→提示已提交
8. 内容哈希输入：ipfs://QmFinalReal
9. 摘要：链上完整方案
10. 点击“验收”→提示“已验收并铸造NFT”（链上accept+mint触发）
11. 点击“列出NFT”→表格出现新NFT记录

### 4. 仪表板
1. 点击“仪表板”
2. 查看用户数、资料数、需求数、NFT数统计
3. 查看链上事件流表格（事件监听实时写入）

### 5. PWA
1. 浏览器DevTools→Application→Service Workers→应看到激活状态
2. 网络离线后刷新页面→应仍能加载核心界面

## 完成标准
- 所有步骤无报错
- 链上事件文件 backend/data/chain_events.json 出现对应事件
- NFT列表展示最新铸造记录
- 学习进度图表随评估更新

✅ 即视为手动端到端验证通过
